5.6 DATABASE DESIGN

A well-structured database design is crucial for efficient data management and retrieval in any application. The database for the "Yoga Master" project is designed to store and manage data in an organized manner, ensuring data integrity, integration, and independence. The main objectives of the database design are:

  • Data Integration: Ensuring that data from various sources is combined into a unified view.
  • Data Integrity: Maintaining accuracy and consistency of data throughout its lifecycle.
  • Data Independence: Allowing data to be modified without affecting the application structure.

The database environment allows multiple programs to access and share data, ensuring fast response times, minimal redundancy, and accurate data management. The components of the database include:

  • File Manager: Manages the storage and retrieval of data.
  • DDL Compiler: Compiles data definition language statements.
  • Database Manager: Controls data access, manipulation, and integrity.

Normalization is applied to the database to eliminate redundancy and ensure logical data dependencies. The steps include:

  1. First Normal Form (1NF): Ensures each column holds a single value and each row is uniquely identifiable.
  2. Second Normal Form (2NF): Ensures all non-key attributes are dependent on the primary key.
  3. Third Normal Form (3NF): Ensures non-key attributes are independent of each other.

By following these normalization procedures, the "Yoga Master" database is structured efficiently, reducing redundancy and enabling quick, accurate data searches.
